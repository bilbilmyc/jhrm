# 弟子 (Disciple) 设计

> **Date**: 2026-06-25
> **Status**: DEFERRED to 元婴+ (per `docs/decisions.md` 决定 #15：字段保留 forward-compat，收徒/培养/派遣 延后实现)
> **Unlock**: 元婴期

---

## 1. 概述

弟子是玩家收的徒弟，培养后可派遣执行探索、战斗等任务。元婴期解锁（玩家自己足够强才能收徒）。候选弟子手写 5-10 个，每个有独立性格、灵根、道心。

---

## 2. 数据模型

```dart
class Disciple {
  String id;
  String name;
  Gender gender;
  Element root;                 // 灵根
  HeartPath heart;              // 初始道心
  int cultivation;              // 修为
  DiscipleRank rank;
  DiscipleStatus status;        // 当前状态
  String currentTask;
  List<String> learnedTechniques;
  List<String> inventoryIds;
}

enum DiscipleRank { 外门, 内门, 真传, 闭关 }
enum DiscipleStatus { 空闲, 修真中, 探索中, 战斗中, 受伤 }
```

## 4. 收徒流程

- 触发：元婴期 + 特定节点（山门、坊市、宗门遗址）
- 流程：候选弟子 IF 段 → 玩家选择 [收为徒] 或 [婉拒]
- 收为徒：加入弟子列表，rank = 外门，cultivation = 0
- 婉拒：候选弟子消失或转为普通 NPC

---

## 5. 培养

### 5.1 修真（自动涨修为）

- 弟子进入「修真中」状态
- 时长按 ADR-0007 修真 ladder：玩家元婴期对应 10 分钟/次
- 每次修真修为 +1 层进度
- 修真过程中可被紧急事件打断

### 5.2 传授功法

- 玩家将已学功法传授给弟子
- 弟子修为 +10
- 弟子获得功法专属加成

### 5.3 赐宝

- 玩家将装备、丹药、灵兽赐给弟子
- 装备提升弟子战斗属性
- 丹药临时恢复状态
- 灵兽陪伴探索

---

## 6. 派遣

### 6.1 探索

- 弟子探索特定节点（灵草谷、妖兽森林、古修遗府等）
- 时长：1-7 天（修真 ladder 时间）
- 收获：灵石、装备、情报
- 派遣上限：同时 1 个弟子探索

### 6.2 战斗

- 弟子参与战斗协助玩家
- 弟子属性独立计算
- 弟子受伤进入「受伤」状态，需休养

### 6.3 任务链

- 元婴期后解锁任务链系统
- 弟子完成多步任务获大奖励
- 任务链跨多个位面

---

## 7. 弟子修真 ladder

| 阶 | 触发条件 | 修真效果 |
|---|---|---|
| 外门 → 内门 | 修为达标 + 师徒 IF 段 | 解锁内门功法、任务 |
| 内门 → 真传 | 修为达标 + 传承 IF 段 | 解锁真传功法、亲传任务 |
| 真传 → 闭关 | 修为达标 + 突破 IF 段 | 独立修炼，可独当一面 |

闭关弟子不需要玩家指挥，按自身道心自动修炼。

---

## 8. UI

**弟子列表页**：每个弟子一行，显示名字、灵根、当前状态、修真进度。

**弟子详情页**：
- 立绘 + 名字 + 灵根 + 道心
- 修为进度条
- 已学功法列表
- 装备 / 灵兽
- 修真历史
- 按钮：[传授] [赐宝] [派遣] [召回]
