# ADR-0002: 金手指 = 隐藏手势 / 密语（开发工具，不是游戏功能）

金手指菜单只能通过**隐藏手势**（如左上角连点 5 下）或**密语**（如输入 `godmode`）调出。正常游戏 UI 中**不**显示任何金手指入口。

## 为什么

这是**个人玩**游戏——开发者就是玩家。金手指是给"作者/玩家"自己的开发工具，不是给"游戏里主角"的能力。如果做成屏幕常驻的金色按钮，会破坏修真世界的 fiction；做成剧情 NPC 给予又增加写作负担。隐藏起来最干净：游戏 UI 完整，金手指用得到。

## 考虑过的方案

- **顶部常驻金手指按钮** — 易找但破坏沉浸感，每次切界面都看到
- **剧情内 NPC 给予**（老爷爷/转世记忆/天书）— 最"修仙"调性，符合网文套路，但要多写一套"获得金手指"的 IF 段
- **隐藏手势/密语** — 玩家不发现就不知道，dev 自己用 — 干净

## 金手指菜单范围（全 4 类）

| 类 | 功能 |
|---|---|
| 修为 / 跳章 | 修为 ×10 / ×100 / 满；跳到下一章；跳过当前战斗/事件 |
| Build / 状态 | 灵根重置；切换道心；调整因果；延长寿元 |
| 一键装备/丹药/灵兽 | 装备最强；丹药满库存；灵兽满好感 |
| 渡劫 / 触发 / 解锁 / 重置 | 强制渡劫成功；强制触发事件；解锁全部位面/秘境；重置存档 |

## 后果

- 不知道手势的玩家无法调出金手指。可接受——这是个人游戏
- 金手指菜单必须**完整**（4 类全覆盖），因为它取代了 dev 的"重玩—测试—调参"循环
- 未来如果上线给朋友玩，要么加一个"教程彩蛋"提示，要么留作"复活节"
